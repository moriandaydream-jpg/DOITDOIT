# 전체 디버깅 결과

## 수정한 주요 문제

- Edge Function 설정 저장 시 `settings` 중복 선언으로 발생한 초기화 오류
- Edge Function JWT 검사로 인한 `인증 헤더 누락` 안내
- 관리자 토큰과 카카오 세션이 함께 있을 때 잘못된 인증 헤더가 전송되던 문제
- 만료된 카카오 토큰이 손님 요청으로 조용히 처리되던 문제
- 익명 사용자가 비어 있는 관리자 자리를 선점할 수 있던 RLS 조건
- 조회수 API 실패 시 게시글 창까지 열리지 않던 문제
- 재연결 후 Auth/Realtime 구독이 중복되거나 끊기던 문제
- 오래된 브라우저 연결 설정이 배포 `config.js`를 덮던 문제
- 관리자 토큰으로 작성한 글이 손님으로 표시되던 문제
- 프록시 게시판 수정이 INSERT 충돌을 일으킬 수 있던 문제
- 설치 파일 삭제 후 설치 링크가 남던 문제
- JSON이 아닌 서버 오류에서 원인이 가려지던 문제
- CORS 주소 끝의 `/` 때문에 GitHub Pages 요청이 막힐 수 있던 문제

## 자동 검사

- 프론트엔드 로컬 모드 스모크 테스트: 8개 통과
- Supabase Edge Function 모의 API 테스트: 10개 통과
- `app.js`, `install.js`, 정리 스크립트 문법 검사 통과
- Supabase Edge Function 및 Cloudflare Worker 문법 검사 통과
- HTML ID 누락/중복 검사 통과
- CSS 괄호 검사 통과
- SQL 정책 생성/삭제 개수 일치 확인

실제 카카오 OAuth 왕복과 원격 Supabase DB 실행은 프로젝트 계정 설정이 필요하므로 로컬 자동 검사 범위에는 포함되지 않습니다.
