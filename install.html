기본 설정

1. Supabase SQL

Supabase SQL Editor에서 이 SQL을 실행하면 게시판, 댓글, 관리자 설정, RLS 정책이 만들어져.

2. config.js

배포할 때 이 내용으로 `config.js`를 교체하면 방문자가 같은 설정을 써.

3. 운영 전환

정리 스크립트
  • DB 비밀키를 숨기려면 Supabase Edge Function 프록시를 쓰고 `config.js`에는 `apiBaseUrl`을 넣어.
  • 카카오 로그인까지 쓰려면 Supabase URL과 publishable key도 Auth 전용으로 같이 넣어.
  • SQL 실행 후 `공유 설정 저장`을 누르면 Supabase에 관리자 ID와 기본 설정이 들어가.
  • 설치가 끝나면 `node remove-installer.js`를 실행해서 설치 파일을 배포본에서 빼.
  • 브라우저 JS는 서버 파일을 직접 삭제할 수 없어서 Node 정리 스크립트를 제공해.
  • 배경은 관리자 화면이나 `config.js`의 `backgroundUrl`로 나중에 바꿀 수 있어.

4. 소셜 로그인

  • 직접 연결 모드와 프록시 모드 모두 카카오 로그인을 쓸 수 있어.
  • Supabase Dashboard > Authentication > Providers에서 Kakao Provider를 켜.
  • Supabase URL Configuration의 Site URL과 Redirect URLs에 위 `로그인 후 돌아올 주소`를 똑같이 넣어.
  • Kakao Developers의 Redirect URI에는 사이트 주소가 아니라 Supabase의 `https://프로젝트.supabase.co/auth/v1/callback`을 넣어.
  • 카카오 로그인은 관리자 전용이며, 관리자 User ID와 다른 계정은 손님으로 돌아가.
  • 로그인 후 관리자 패널에서 `내 ID 사용`과 `공유 설정 저장`을 누르면 오너가 고정돼.
